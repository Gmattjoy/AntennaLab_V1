PROJECT
AntennaLab V1

PROJECT ROLE
AntennaLab V1 is an Android-based antenna design, analysis, testing,
and project-management application.

The application supports both novice antenna builders and more advanced
radio operators by combining guided antenna design with engineering-style
analysis and measurement tools.

The system integrates:

• antenna design wizard
• calculation engine
• project workspace
• measurement tools
• antenna testing workflows
• tuning assistance
• hardware-aware measurement systems

The project is currently being built for Android phones and tablets,
with architectural consideration for larger-screen layouts and possible
future desktop-style workflows.

This file acts as the **master project context document** and is intended
to allow future AI sessions to understand the project quickly and safely.

########################################################################
HARDWARE CAPABILITY ARCHITECTURE
########################################################################

The application uses a capability-based hardware system.

Hardware selection stored in ProjectData:

testHardwareProfile

This resolves to a structured capability profile which controls:

• measurement types available
• graph display availability
• marker system availability
• delta marker capability
• Smith chart support
• S21 estimate support
• cable fault / TDR preview support
• CSV export support
• sweep frequency limits

This approach prevents UI code from hardcoding hardware behaviour.

Instead:

Hardware Profile
        ↓
Capability Profile
        ↓
Testing UI features

This architecture allows easy support for additional hardware later.

CURRENT SWEEP TESTING SYSTEM
########################################################################

SweepGraphScreen provides the first unified measurement interface.

Current capabilities include:

• SWR graph
• return loss graph
• resistance graph
• reactance graph
• Smith chart preview
• impedance locus display
• S21 estimate preview
• marker A and marker B
• delta measurement readout
• resonance detection
• cable fault / TDR preview estimate
• CSV export preview

These functions operate on the SweepResult model.

########################################################################
PROJECT DEVELOPMENT PROTOCOL
########################################################################

This project uses controlled AI-assisted development.

Rules:

• full file replacements only
• instruction section separated from file content
• use visible Android Studio tree exactly
• preserve code comment structure
• do not introduce architectural drift
• avoid incomplete refactor states
• provide follow-up edits when compile issues are expected
• prioritize architecture stability over rapid changes

The human operator performs file creation and pasting.
The AI provides structured instructions.

########################################################################
PROJECT FOUNDATION RULE
########################################################################

The project should focus on building the application while also
laying a foundation for more advanced engineering software later.

Guidelines:

• avoid large refactors during active feature development
• improve structure gradually when files are already being edited
• prefer reusable domain logic over screen-bound logic
• preserve architecture boundaries
• treat source code as a long-term engineering asset

Goal:

Build the working application while preparing the architecture
for future professional antenna engineering capabilities.
